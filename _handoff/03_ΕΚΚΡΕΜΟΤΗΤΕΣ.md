# 03 — ΕΚΚΡΕΜΟΤΗΤΕΣ

## Α) Ανάπτυξη στο TEST (μπορούμε να συνεχίσουμε τώρα)
- **Σύγκριση vaicon-eidikes ↔ vaicon-app** για να βρεθεί τι λειτουργίες υπάρχουν στις ειδικές και ΔΕΝ έχουν περαστεί ακόμα στις τυποποιημένες. (Αυτό είναι το συμφωνημένο επόμενο βήμα — βλ. αρχείο 05.)
- **Κοινή βάση (Β):** η *υλοποίηση/λογική* γράφεται στο test. Το πραγματικό "κούμπωμα" γίνεται στο go-live.
- **Installations ↔ app (Γ):** ίδια λογική — δουλεύει ήδη στο test, χρειάζεται οριστικοποίηση στο go-live.

## Β) Εργασίες GO-LIVE (παρκαρισμένες μέχρι να τελειώσουμε)
- **Auth στην παραγωγή του vaicon-app:** τώρα `useAuth=false` στο PROD. Πριν ανοίξει:
  1. Δημιουργία χρηστών στη Firebase Auth της `vaiconcloud` (ίδιοι με test).
  2. Αλλαγή `useAuth: true` στο PROD block του `firebaseConfig.js`.
  Αλλιώς δεν μπορεί να μπει κανείς. Μέχρι τότε μηνύματα/χρήστες/guest ΔΕΝ λειτουργούν live.
- **Versioning στο vaicon-installations:** ✅ στήθηκε (`scripts/bump-version.js` τοπικού μετρητή + `version.js` + εμφάνιση έκδοσης στην κεφαλίδα). Πριν το πρώτο deploy τρέχει `node scripts/bump-version.js` όπως στα άλλα δύο.
- **Τελικό migration:** όταν τελειώσει η ανάπτυξη →
  1. Παίρνουμε τα δεδομένα παραγωγής (`vaiconcloud` + `vaicon-eidikes`),
  2. τα ενσωματώνουμε στο test (αφού καθαριστεί),
  3. το test γίνεται η νέα παραγωγή — χωρίς να το καταλάβει κανείς.

## Γ) Προϋπάρχουσες εκκρεμότητες GIT (ενημερωτικά — να γίνουν ΑΡΓΟΤΕΡΑ)
Κατάσταση τη στιγμή της παράδοσης (13/06/2026). ΔΕΝ έγινε push/commit — αφέθηκαν σκόπιμα.
- **vaicon-app:** καθαρό & pushed. Μόνο ο φάκελος `_handoff/` είναι untracked (προσωρινός).
- **vaicon-eidikes:** 1 commit χωρίς push → `chore: auto-bump version` (αβλαβές, μόνο έκδοση).
  Επίσης untracked το νέο `.cursor/rules/versioning.mdc`.
- **vaicon-installations:** 1 commit χωρίς push → "backup: dev εργασία — dev polling, ανάγνωση
  και των δύο εργοστασίων, badge factory" + **6 αρχεία μη committed** (App.js, AppointmentsScreen.js,
  CalendarScreen.js, OrderDetail.js, OrdersScreen.js, fbUtils.js). Επίσης untracked το `.cursor/`.

⚠️ ΠΡΟΣΟΧΗ πριν push σε eidikes/installations: είναι ΖΩΝΤΑΝΑ/σε χρήση. Οι αλλαγές του
installations είναι μισοτελειωμένη δουλειά ενοποίησης (ανάγνωση και των δύο εργοστασίων) —
ΜΗΝ γίνουν push πριν ολοκληρωθεί η κοινή βάση, αλλιώς βγαίνουν live σε πραγματικούς χρήστες.
Όλα είναι ασφαλή στον δίσκο· θα τα χειριστούμε όταν το αποφασίσει ο χρήστης.
