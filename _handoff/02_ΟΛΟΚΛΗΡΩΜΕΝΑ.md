# 02 — ΟΛΟΚΛΗΡΩΜΕΝΑ (στο vaicon-app, δοκιμασμένα στο test)

## Μηνύματα & χρήστες
- Σύστημα μηνυμάτων admin↔users **κοινό με vaicon-eidikes** (ίδιοι users/admin, κοινά μηνύματα).
- `MessagesScreen.js`: forced-read queue — popup ανά 5' μέχρι να ανοιχτούν, αναδύεται το παλιότερο, μένει μέχρι "διαβάστηκε", μετά το επόμενο.
- Read receipts ("διαβάστηκε") για μηνύματα admin.
- Ταυτότητα χρήστη/ρόλοι (`currentUser`, `APP_USERS`, `lockKey`, `userFromEmail`, `user_labels`).
- Εμφάνιση ονόματος χρήστη που έκανε καταχώρηση (admin view) — με όνομα, όχι μόνο USER id.
- **Όνομα τρέχοντος χρήστη στο header** σε πλαίσιο, κόκκινο φόντο, δεξιά από το version.
- Admin "Διαχειριστής": ξεκλείδωμα μία φορά (όχι κάθε φορά) + κουμπί κλειδώματος.

## Περιορισμοί guest
- Read-only, περιορισμένη ορατότητα (μόνο προς κατασκευή / παραγγελίες / έτοιμα αποθήκης, μονή & διπλή).

## Ειδοποιήσεις πελατών (SMS / Viber / Email) — όπως vaicon-eidikes
- Netlify functions: `netlify/functions/` → `send-sms.js`, `send-viber.js`, `dlr-callback.js`, `unsubscribe.js`, `lib/fbAdmin.js` (Yuboto API).
- `netlify.toml`: `functions = "netlify/functions"`, esbuild bundler.
- Στήλη κουμπιών (Viber/Email/SMS) **τέρμα δεξιά** στις κάρτες — πάντα ορατά, αλλά **ανενεργά/γκρι** όταν λείπει στοιχείο επικοινωνίας (ομοιομορφία).
- Επιβεβαίωση πριν την αποστολή (`confirmSend`).
- Πεδία πελάτη: `phone2`, `phone3`, `phoneViber`, `email` (+ viberOptOut) στο `CustomersScreen.js`.
- ⚠️ Οι Netlify functions τρέχουν ΜΟΝΟ στο deployed site, όχι τοπικά. Default `YUBOTO_TEST_MODE=true` (δεν στέλνει αληθινά αν δεν μπουν env vars).

## Σύστημα επενδύσεων (coatings) — Φάσεις Α/Β/Γ από vaicon-eidikes
- `formatHelpers.js` (αντιγραφή από eidikes): bold/size/color στις επενδύσεις.
- **Φάση Α:** μορφοποίηση στο `CoatingsScreen.js` + styled ονόματα στις κάρτες.
- **Φάση Β:** `coatingDetails` ανά επένδυση (auto διαστάσεις από h/w), κουμπί "i" + modal λεπτομερειών, ενσωμάτωση στη φόρμα.
- **Φάση Γ:** προειδοποίηση ματιού (αλουμίνιο/κυπρί ή σχετικές σημειώσεις) → modal επιβεβαίωσης στην αποθήκευση, με δυνατότητα προσθήκης σημείωσης "ΠΡΟΣΟΧΗ ΟΧΙ ΤΡΥΠΗΜΑ ΓΙΑ ΜΑΤΙ".
- Η σημείωση εμφανίζεται **κόκκινη/bold/μεγαλύτερη** σε κάρτες (`renderNotesWithWarning`) και εκτυπώσεις (`notesHtmlWithWarning` στο `printUtils.js`).

## Πεδίο τοποθέτησης
- Πεδίο "Τοποθέτηση" στις παραγγελίες vaicon-app + badges (μοντάρισμα/τοποθέτηση) στις κάρτες, με ενσωμάτωση στο installations. Phase buttons στη μεσαία στήλη της κάρτας (οπτικά κοντά στο eidikes).

## Ανθεκτικότητα φόρτωσης
- `firebaseRealtime.js`: timeout 12s + error callbacks στα onValue ώστε να μην κολλάει το spinner.

## Σύστημα Πωλητών & Εγκρίσεων (18/6/2026 — app + eidikes + installations)
Λογαριασμοί `seller1`…`seller5@vaicon.local` (ρόλος user, πρόθεμα `seller`). Firebase/κανόνες: βλ. 07.

Περιορισμοί/ορατότητα πωλητή:
- Βλέπει μόνο παραγγελίες των δικών του πελατών (τυποποιημένες, ειδικές, τοποθετήσεις — κάρτες/ημερολόγιο/ραντεβού).
- Κρυφά: εκτυπώσεις, Μενού, αρχείο, δημιουργία νέου πελάτη, αριθμός παραγγελίας & αριθμός προγράμματος (στις ειδικές).
- Μενού (☰) πωλητή: μόνο «Ανανέωση» + «Αποσύνδεση».

Ανάθεση πελάτη→πωλητή:
- Πεδίο/dropdown πωλητή στην καρτέλα πελάτη (το θέτουν απλοί χρήστες, όχι πωλητές/guest).
- Καταγραφή στο `seller_assign_log` → οθόνη `SellerLogScreen.js` (admin-only).
- Φίλτρο πωλητή: στους πελάτες + στην αριστερή μπάρα (app & eidikes) δείχνει μόνο τα δικά του.

Ροή έγκρισης:
- Υποβολή πωλητή → `seller_submissions` ΧΩΡΙΣ αριθμό (`CustomScreen.js` / `SpecialScreen.js`).
- `ApprovalScreen.js`: Έγκριση (δίνει αριθμό, περνά σε std_orders/special_orders, log) ή Απόρριψη (σημείωση).
- `SellerSubmissionsScreen.js` («Οι υποβολές μου», κουμπί στην αριστερή μπάρα): pending/rejected, Διαγραφή + Διόρθωση (prefill) στις απορριφθείσες, επισύναψη εγγράφου.
- `ApprovalHistoryScreen.js`: εγκρίσεις + απορρίψεις, auto-καθαρισμός 4 μηνών.
- Δικαιώματα έγκρισης (`approval_rights`) τα ορίζει ο admin.
