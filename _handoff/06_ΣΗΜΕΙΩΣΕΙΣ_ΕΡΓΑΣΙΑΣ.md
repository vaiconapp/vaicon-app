# 06 — ΣΗΜΕΙΩΣΕΙΣ ΕΡΓΑΣΙΑΣ (gotchas)

## Τοπικό dev server
- **Μην τρέχεις και τα 3 προγράμματα ταυτόχρονα** στο localhost: εξαντλούνται οι συνδέσεις του browser και κολλάει η φόρτωση (λευκή σελίδα/spinner). Αν κολλήσει, κλείσε τα άλλα — ανεβαίνουν μόνα τους.
- Μετά από αλλαγές: hard refresh **Ctrl+Shift+R**.
- Σε Expo Web το `Alert.alert` με πολλά κουμπιά δεν δουλεύει σωστά → χρησιμοποιούμε **Modal overlay** αντί για Alert.

## Netlify functions (SMS/Viber/Email)
- Τρέχουν ΜΟΝΟ στο deployed site, ΟΧΙ στο τοπικό dev. Τοπικά τα κουμπιά αποστολής "αποτυγχάνουν" — αναμενόμενο.
- Default `YUBOTO_TEST_MODE=true`: δεν στέλνει αληθινά μηνύματα αν δεν μπουν τα env vars.

## Ambiguous Unicode warning στον editor
- Είναι καλοήθης προειδοποίηση λόγω ελληνικών σε labels — ΟΧΙ bug. Αγνοείται.

## Κοινά δεδομένα στο test
- Πελάτες/επενδύσεις/κλειδαριές είναι κοινά στο `vaicon-test`, οπότε αλλαγή σε ένα πρόγραμμα φαίνεται και στα άλλα (στο test). Rename πελάτη διαδίδεται, delete protection όταν υπάρχουν παραγγελίες.

## Ροή αλλαγών & επίδραση στην παραγωγή (ΑΝΑ ΠΡΟΓΡΑΜΜΑ)
Ισχύει για ΚΑΙ ΤΑ ΤΡΙΑ προγράμματα:
- Δουλεύεις τοπικά → η εφαρμογή χρησιμοποιεί τη βάση `vaicon-test`. Τα δεδομένα test
  ΔΕΝ μπλέκονται ΠΟΤΕ με τα δεδομένα παραγωγής (γράψε/σβήσε/μετάφερε ελεύθερα).
- Ο ΚΩΔΙΚΑΣ βγαίνει ζωντανά ΜΟΝΟ με `git push` (Netlify auto-deploy). Μέχρι τότε, τίποτα live.
- Το μεγάλο "πέρασμα δεδομένων" (migration) γίνεται ΧΕΙΡΟΚΙΝΗΤΑ στο τέλος, ποτέ τυχαία.

⚠️ ΚΡΙΣΙΜΗ διαφορά στην κατάσταση παραγωγής:
| Πρόγραμμα | Δεδομένα test↔live | Push επηρεάζει ενεργούς χρήστες; |
|---|---|---|
| vaicon-app | Δεν μπλέκονται | ΟΧΙ — "νεκρό", κανείς δεν το χρησιμοποιεί τώρα |
| vaicon-eidikes | Δεν μπλέκονται | **ΝΑΙ — άμεσα** (ζωντανό, σε χρήση στο εργοστάσιο) |
| vaicon-installations | Δεν μπλέκονται | **ΝΑΙ — άμεσα** (ζωντανό, σε χρήση στο εργοστάσιο) |

Συνέπεια: αλλαγές στα eidikes/installations → δοκίμασέ τες ΚΑΛΑ τοπικά και κάνε push
σε ήσυχη ώρα (λάθος κώδικας φαίνεται αμέσως σε πραγματική χρήση — τα ΔΕΔΟΜΕΝΑ όμως
παραμένουν ασφαλή). Στο installations: στήσε versioning ΠΡΙΝ το πρώτο push (βλ. αρχείο 04).
